  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

    2. Internet es la red física de computadoras conectadas globalmente. La World Wide Web (WWW) es uno de los servicios que funciona sobre Internet y permite acceder a páginas web mediante navegadores.

  2. ¿Cuáles son las partes de un URL?

    3. Un URL (ej. https://www.ejemplo.com:443/ruta?query=valor) se compone de:

    • Protocolo (https)
    • Nombre del dominio (www.ejemplo.com)
    • Puerto (443, opcional)
    • Ruta (/ruta)
    • Parámetros de consulta (?query=valor)
  3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    4. GET: Solicita datos
    HEAD: Solicita solo cabeceras
    POST: Envía datos nuevos
    PUT: Reemplaza datos existentes
    PATCH: Modifica parcialmente
    DELETE: Elimina recursos

  4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    5. Se debe usar POST porque los datos se envían de forma segura sin aparecer en la URL.

  5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    6. Se utiliza el método GET.

  6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    7. Significa que todo salió bien. No hubo errores.

  7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    8. Sí, porque indica que el recurso no fue encontrado. Puede haber un enlace roto o un archivo faltante.

  8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    9. Sí. El error 500 indica una falla en el servidor, generalmente en el código, y debe ser revisado por el desarrollador.

  9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    10. Significa que ya no se recomienda su uso y podrían eliminarse en futuras versiones. Ejemplos: <font>, <center>, <big>.

  10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?
    • HTML5 incluye etiquetas semánticas como <header>, <footer>, <article>.
    • Soporte para multimedia con <video> y <audio>.
    • Validación nativa en formularios.
    • API modernas como geolocalización, canvas, etc.
  11. ¿Qué componentes de estructura y estilo tiene una tabla?
    • Estructura: <table>, <tr>, <th>, <td>, <caption>
    • Estilo: se aplica con CSS (bordes, padding, alineación, etc.)
  12. ¿Cuáles son los principales controles de una forma HTML5?
    • <input type="text">, <email>, <password>, <date>, <range>, etc.
    • <textarea>
    • <select> y <option>
    • <button>
  13. ¿Qué tanto soporte HTML5 tiene el navegador que utilizas?

    14. Uso Google Chrome y en la prueba de html5test.com obtuve 528 de 555 puntos. Tiene un soporte casi completo.
